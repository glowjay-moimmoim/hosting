Why Lovable

코드 몰라도 됩니다

기존에는 상세페이지 하나 만들려면 디자이너에게 의뢰하거나, 직접 코딩을 배워야 했습니다.
Lovable은 텍스트로 요청하면 전체 페이지를 완성된 형태로 뽑아줍니다.

피그마 디자인 → 개발자 코딩 → 수정 반복
↓ Lovable
프롬프트 입력 → 완성된 웹페이지 → URL 공유

Session Flow

오늘 강의 흐름

1

오늘 만들 것 미리 보기

완성된 상세페이지를 먼저 보고 시작합니다. 오늘 뭘 만드는지 눈으로 확인하고 나면 방향이 잡힙니다.

2

왜 Lovable인가

코드 없이 완성 페이지가 나오는 원리. 기존 방식과 어떻게 다른지 비교합니다.

3

디자인 시스템이 뭔가요

컬러, 폰트, 버튼 스타일을 문서로 정리한 것. 이걸 Lovable에 넣으면 일관된 스타일로 만들어줍니다.

4

상세페이지 기획안 작성법

히어로 / 문제제기 / 솔루션 / 사회적 증거 / CTA — 섹션별로 무엇을 담을지 텍스트로 정리하는 법.

5

🔑 Lovable에 프롬프트 넣는 법

디자인 시스템 + 기획안 + 요청사항을 어떻게 구성해서 넣는지. 실제 프롬프트 예시 그대로 따라합니다.

6

결과 다듬기

마음에 안 드는 부분만 좁혀서 수정 요청하는 프롬프트 패턴. "이 부분만 바꿔줘" 식으로 요청합니다.

7

배포 & 공유

Lovable에서 URL 뽑는 법. 강의가 끝나면 링크 하나가 생깁니다.

Prompt Example

이렇게 넣으면 됩니다

디자인 시스템 + 기획안을 같이 넣으면 훨씬 빠르게 원하는 결과가 나옵니다.

PROMPT STRUCTURE

[디자인 시스템] - 배경색: #fafaf8 - 강조 컬러: #eb5a0f (주황) - 폰트: Pretendard - 카드 모서리: 12px, 그림자 없음 [기획안] 히어로: "코드 없이 상세페이지를 만드는 법" 문제제기: 디자이너 없이 페이지 만들기 어렵다 솔루션: Lovable로 텍스트만 넣으면 완성 CTA: "지금 바로 만들어보기" 위 내용으로 상세페이지 HTML을 만들어줘. 모바일 반응형으로 해줘.
